# March 31, 2026 Node Package Incident

## Default targets

- `axios@1.14.1`
- `axios@0.30.4`
- `plain-crypto-js@4.2.1`

## Why these versions are defaults

These defaults are based on the March 31, 2026 incident where the npm registry metadata showed publish timestamps for those versions, while the current registry state no longer served them as normal package versions.

Primary sources checked during skill creation:

- `axios` registry metadata: [https://registry.npmjs.org/axios](https://registry.npmjs.org/axios)
- `plain-crypto-js` registry metadata: [https://registry.npmjs.org/plain-crypto-js](https://registry.npmjs.org/plain-crypto-js)
- Official axios tags: [https://github.com/axios/axios/tags](https://github.com/axios/axios/tags)

Useful details from those sources at creation time:

- `axios` showed time entries for `1.14.1` and `0.30.4`, but the active `versions` set no longer contained them.
- `axios` tags included `v1.14.0` and `v0.30.3`, but not `v1.14.1` or `v0.30.4`.
- `plain-crypto-js` showed time entries for `4.2.0` and `4.2.1`, while the package later pointed to `0.0.1-security.0`.

## Updating the target list

When the incident scope changes:

1. Update `DEFAULT_TARGETS` in `scripts/scan_compromised_versions.py`.
2. Update this reference file to match the new target set and sources.
3. Re-run the script on a small fixture with positive and negative cases.
