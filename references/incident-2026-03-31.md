# Supply-Chain Compromise of axios npm Package

**Date**: 2026-03-31
**Severity**: Critical
**Status**: Active compromise
**Discovered by**: Elastic Security researchers

## Summary

An npm maintainer account (`jasonsaayman`) on the widely used `axios` HTTP client package was compromised. The attacker published two malicious versions of `axios` and a new typosquat package, `plain-crypto-js`, which served as the payload delivery vehicle. At the time of discovery, both the `latest` and `legacy` dist-tags pointed to compromised versions, so default `npm install axios` resolved to a backdoored package.

The malicious dependency `plain-crypto-js` contains an obfuscated `postinstall` script that downloads and executes platform-specific stage-2 payloads from an external C2 server on macOS, Windows, and Linux. The payload self-deletes and overwrites its own `package.json` to reduce obvious post-incident traces.

## Who Is Affected

Any system that ran `npm install` or an equivalent install resolving `axios@1.14.1` or `axios@0.30.4` after `2026-03-31T00:21:58Z` may have executed the stage-2 payload. This includes:

- Developer workstations
- CI/CD pipelines
- Production deployments pulling fresh installs
- Any project with `axios` as a direct or transitive dependency without a lockfile pinning to a prior safe version

## Affected Packages

- `axios@1.14.1`: malicious, tagged `latest` at time of discovery
- `axios@0.30.4`: malicious, tagged `legacy` at time of discovery
- `plain-crypto-js@4.2.0`: malicious, payload delivery vehicle
- `plain-crypto-js@4.2.1`: malicious, payload delivery vehicle

## Safe Versions

- `axios@1.14.0`: last legitimate 1.x release, published via GitHub Actions OIDC with SLSA provenance
- `axios@0.30.3`: last legitimate 0.30.x release

## Recommended Actions

1. Check lockfiles and installed package metadata for `axios@1.14.1`, `axios@0.30.4`, `plain-crypto-js@4.2.0`, and `plain-crypto-js@4.2.1`.
2. If any default target is present on a real system, treat the host or environment as potentially compromised.
3. Pin to safe versions such as `axios@1.14.0` or `axios@0.30.3`.
4. Rotate credentials on any system where the compromised packages were installed.
5. Block the C2 domain `sfrclak.com` at the network perimeter.

## Indicators of Compromise

### Network

- C2 server: `http://sfrclak.com:8000/`
- Campaign ID: `6202033`
- Full C2 URL: `http://sfrclak.com:8000/6202033`

### Filesystem

- macOS stage-2 binary: `/Library/Caches/com.apple.act.mond`
- Windows renamed PowerShell copy: `%PROGRAMDATA%\wt.exe`
- Windows transient VBScript loader: `%TEMP%\6202033.vbs`
- Windows transient PowerShell payload: `%TEMP%\6202033.ps1`
- Linux stage-2 Python script: `/tmp/ld.py`

### npm Account

- Compromised account: `jasonsaayman`
- Attacker email: `ifstap@proton.me`
- Legitimate email before compromise: `jasonsaayman@gmail.com`

## Evidence of Account Compromise

- `axios@1.14.0` was published via GitHub Actions OIDC trusted publishing with provenance attestations.
- `axios@1.14.1` was published via direct CLI publish, without OIDC or provenance, and added `plain-crypto-js@^4.2.1` as the only dependency-level change from `1.14.0`.
- `axios@0.30.4` was published about 39 minutes later and likewise added `plain-crypto-js@^4.2.1` to the otherwise legitimate `0.30.3` release.
- The maintainer email changed from `jasonsaayman@gmail.com` to `ifstap@proton.me` on the malicious versions.

## Delivery Chain

1. Installing `axios@1.14.1` or `axios@0.30.4` pulls `plain-crypto-js@^4.2.1`.
2. `plain-crypto-js` declares `postinstall: node setup.js`.
3. `setup.js` is an obfuscated multi-platform backdoor installer.

## Obfuscation

- Layer 1: string reversal plus base64 decoding
- Layer 2: XOR using key `OrDeR_7077` with position-dependent index `7 * i^2 % 10`

Decoded string highlights from the reported payload:

- `child_process`
- `os`
- `fs`
- `http://sfrclak.com:8000/`
- `win32`
- `darwin`
- `package.json`
- `package.md`

## Platform-Specific Behavior

### macOS

- Writes an AppleScript to `$TMPDIR/<campaign_id>`
- Downloads a binary to `/Library/Caches/com.apple.act.mond`
- Sets permissions and executes it in the background via `/bin/zsh`
- Uses `osascript` for process-tree evasion

### Windows

- Locates PowerShell via `where powershell`
- Copies it to `%PROGRAMDATA%\wt.exe`
- Writes a transient `.vbs` loader to `%TEMP%`
- Downloads a `.ps1` payload from the C2 and executes it hidden
- Deletes the `.ps1` and `.vbs` after execution

### Linux

- Downloads a Python script to `/tmp/ld.py`
- Executes it in the background via `nohup`

## Anti-Forensics

- `setup.js` removes itself
- A clean `package.md` is renamed to `package.json`, replacing the malicious version and hiding the `postinstall` hook from casual inspection
